- Ruby 3.2.4 / Rails 7.1 / PostgreSQL / TailwindCSS 4.3.0 / Hotwire/Turbo
- 认证：Devise

- 分页：Kaminari
- 测试框架：Minitest

## 一、角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| super_admin | `role: "super_admin"` | 超级管理员 |
| admin | `role: "admin"` | 管理员 |
| member | `role: "member"` | 付费会员 |
| user | `role: "user"` | 普通注册用户 |
| 游客 | 未登录 | 未认证访客 |

所有登录用户（super_admin / admin / member / user）均可收藏和取消收藏股票。游客无法进行任何收藏操作。

## 三、分角色交互场景测试

### 3.1 场景一：普通用户（user）收藏股票

```
1. 用户登录（sign_in）
2. 访问股票详情页（/stocks/:id）
3. 看到蓝色【收藏股票】按钮
4. 点击收藏 → Turbo Stream 局部刷新 → 按钮变为灰色【已关注】
5. 访问个人中心（/users/profile/favorites）
6. 看到新收藏的股票卡片
7. 点击【取消收藏】 → Turbo Stream 局部移除卡片
8. 确认页面不再显示该股票
```

### 3.2 场景二：付费会员（member）查看金字塔评分

```
1. 会员登录
2. 访问 /users/profile/favorites
3. 每个收藏卡片展示金字塔评分（颜色按分数分级）
4. 非会员（user）不显示该字段
```

### 3.3 场景三：管理员（admin）操作

```
1. admin 登录
2. 收藏/取消收藏股票功能同普通用户
3. 管理员可查看自己的收藏列表
4. 管理员不能查看其他用户的收藏列表（Pundit 策略限制）
```

### 3.4 场景四：超级管理员（super_admin）操作

```
1. super_admin 登录
2. 收藏功能完全同普通用户
3. 不能查看其他用户的收藏列表
```

### 3.5 场景五：游客操作

```
1. 未登录访问股票详情页
2. 看到蓝色【收藏股票】按钮（实际为 link_to 跳转登录页）
3. 点击 → 跳转至 /users/sign_in 登录页
4. 直接访问 /users/profile/favorites → 重定向至登录页
5. 直接发 POST/DELETE 请求 → 被 authenticate_user! 拦截
```
